MCU: 2x Arduino Nano/Uno

Связь: RS485 (чип MAX485), Master-Slave архитектура с защитой от разрыва линии.

Память: Внешний EEPROM модуль AT24Cxx по шине I2C (для вечного хранения Тотала тюков и настроек без износа флеша процессора).

Надежность: Аппаратный Watchdog (перезагрузка при зависаниях), неблокирующие задержки (millis()), полная программная фильтрация дребезга контактов.

ТЕХНИЧЕСКАЯ ДОКУМЕНТАЦИЯ СИСТЕМЫ УПРАВЛЕНИЯ ПРЕСС-ПОДБОРЩИКОМ

Версия ПО: 1.0 (Release)
Архитектура: Распределенная система Master-Slave (RS485)

ЧАСТЬ 1. РУКОВОДСТВО ОПЕРАТОРА (Для Тракториста)

1.1. Описание пульта управления

Пульт установлен в кабине трактора и предназначен для контроля процесса обвязки рулонов.
На передней панели расположены:

Тумблер «АВТО / РУЧНОЙ»: Переключение режимов работы пресса.

Тумблер «СЕТКА / ШПАГАТ»: Выбор обвязочного материала (используется также для настроек).

Кнопка «ЭКРАН»: Переключение страниц информации.

Кнопка «ДЕЙСТВИЕ»: Универсальная кнопка (Запуск мотора в ручном режиме / Сброс аварии в автоматическом).

1.2. Режимы работы

АВТОМАТИЧЕСКИЙ РЕЖИМ (Основной)

Переведите тумблер в положение [A] (АВТО).

Начните движение и сбор массы. На экране горит: NABOR MASSY...

При достижении нужной плотности пресс издаст 3 коротких звуковых сигнала. На экране появится: STOP TRAKTOR!.

Остановите трактор. Система выждет заданную паузу и автоматически запустит подачу сетки/шпагата.

После обрезки ножом прозвучат 2 звуковых сигнала (OTKROY DVER!). Выгрузите тюк. Цикл завершен.

РУЧНОЙ РЕЖИМ (С Калибровкой)
Используется при сбоях или для настройки оборудования.

Переведите тумблер в положение [R] (РУЧНОЙ).

При наборе плотности система не запустит мотор сама.

В правом верхнем углу экрана запустится секундомер. Он показывает, сколько времени прошло с момента срабатывания датчика плотности.

Для запуска обвязки нажмите и удерживайте кнопку «ДЕЙСТВИЕ». Секундомер остановится. Запомните эту цифру — её можно будет ввести в меню настроек для идеального автоматического режима!

1.3. Меню настроек

Для входа в настройки необходимо:

Перевести тумблер режима строго в АВТО.

Зажать одновременно кнопки «ЭКРАН» и «ДЕЙСТВИЕ» на 2 секунды. На экране появится == NASTROYKI ==.

Короткими нажатиями кнопки «ЭКРАН» листайте параметры (Плотность, Ожидание трактора, Сетка, Шпагат).

Для изменения секунд используйте тумблер «СЕТКА/ШПАГАТ»:

Переключите тумблер в ШПАГАТ и нажимайте «ДЕЙСТВИЕ» для увеличения (+).

Переключите тумблер в СЕТКА и нажимайте «ДЕЙСТВИЕ» для уменьшения (-).

Для сохранения снова зажмите две кнопки на 2 секунды. Появится надпись SOHRANENO V PRES.

1.4. Нештатные ситуации

Иконка [X] в углу экрана: Обрыв связи между трактором и прессом (поврежден кабель). Пресс перейдет в автономный режим (будет работать по датчикам, но без пульта).

SBOY! PROVER!: Произошла авария (мотор работал слишком долго, или нож не упал). Включится непрерывный писк. Устраните затор в прессе и зажмите кнопку «ДЕЙСТВИЕ» на 1 секунду для сброса аварии.

ЧАСТЬ 2. МОНТАЖНАЯ СХЕМА (Для Электронщика)

2.1. Требования к питанию

Запрещается прямое подключение модулей Arduino к бортовой сети 12-14V трактора (вызовет перегрев LDO-регулятора).

Блок питания: Использовать понижающий DC-DC преобразователь (LM2596/MP1584), настроенный строго на 5.0V.

Защита по входу: На входе 12V установить TVS-диод (супрессор) 1.5KE18A и электролитический конденсатор (1000мкФ 35В) для гашения импульсов генератора.

Защита реле: Параллельно катушке силового автомобильного реле пресса обязательно установить обратный диод (1N4007) полоской к плюсу.

2.2. Распиновка Пульта Управления (Master)

D2: Тумблер «Сетка/Шпагат» (Замыкание на GND = Сетка). Внутренняя подтяжка (PULLUP).

D3: Кнопка «ЭКРАН» (Замыкание на GND). Внутренняя подтяжка.

D4: Тумблер «Авто/Ручной» (Замыкание на GND = Ручной). Внутренняя подтяжка.

D11: Кнопка «ДЕЙСТВИЕ» (Замыкание на GND). Внутренняя подтяжка.

I2C (A4/A5): Дисплей LCD 1602.

D8 (RX) / D9 (TX) / D10 (EN): Модуль связи MAX485.

2.3. Распиновка Блока Пресса (Slave)

Входы (Датчики - NPN типа / Замыкание на GND):

D2: Концевик «Плотность».

D3: Концевик «Старт» (Начало захвата).

D4: Концевик «Конец» (Отсечка ножом).

D5: Концевик «Дверь».

D8: Локальная кнопка «Сброс/Тест».

D11: Локальный резервный тумблер «Сетка/Шпагат».

Выходы (Реле-модули):

D6: Силовое реле мотора (ШПАГАТ). Active HIGH.

D10: Силовое реле мотора (СЕТКА). Active HIGH.

D7: Реле звукового сигнала (Гудок). Active LOW (Инвертированная логика!).

D9: Реле проблескового маячка. Active HIGH.

Шины данных:

A0 (RX) / A1 (TX) / A2 (EN): Модуль связи MAX485.

I2C - A4 (SDA) / A5 (SCL): Внешний модуль памяти EEPROM (AT24Cxx).

Терминация RS485: Между линиями A и B на обеих сторонах магистрали обязательна установка терминальных резисторов 120 Ом (если отсутствуют на самих модулях).

ЧАСТЬ 3. АРХИТЕКТУРА ПО (Для Программиста)

3.1. Парадигма работы

Система работает по принципу жесткого автомата состояний (Finite State Machine) на базе Блока Пресса (Slave). Блок Пульта (Master) является "Тонким клиентом": он не принимает решений об обвязке, а лишь работает как HMI-интерфейс, считывая статус Slave-модуля и отправляя ему конфигурацию.

3.2. Автомат состояний (BalerState)

Ядро логики описано в switch (currentState):

WAIT_DENSITY (0): Ожидание фильтрованного сигнала t_Dens от датчика плотности.

WAIT_TRACTOR (1): Пауза t_Stop для остановки трактора перед подачей сетки.

WAIT_START_SENSOR (2): Контроль начала механического захвата (Таймаут 10с -> Авария).

MOTOR_RUNNING_TIMER (3): Вращение по таймеру t_Net или t_Twine.

WAIT_END_SENSOR (4): Контроль отсечки (Таймаут 15с -> Авария).

WAIT_DOOR (5): Инкремент Тотала и счетчика смены по фронту открытия/закрытия двери.

ERROR_STATE (6): Глобальная блокировка. Сброс только по внешней команде doReset.

TEST_MODE (7): Сервисный режим прозвонки концевиков (выход по таймауту 60с).

3.3. Карта памяти EEPROM (I2C AT24Cxx, Адрес 0x50)

Используется прямой доступ к шине Wire (без сторонних библиотек) для повышения надежности.

0x00 (1 byte): Маркер формата (0x42). Если не совпадает, EEPROM инициализируется дефолтными значениями.

0x01 (4 bytes): totalBales (uint32_t). Общий счетчик тюков.

0x05 (1 byte): t_Dens. Таймер плотности.

0x06 (1 byte): t_Stop. Таймер ожидания.

0x07 (1 byte): t_Net. Таймер сетки.

0x08 (1 byte): t_Twine. Таймер шпагата.

3.4. Отказоустойчивость

Hardware Watchdog: На обеих платах задействован wdt_enable(WDTO_2S). При зависании основного цикла (глухой while, аппаратный сбой шины) контроллер аппаратно перезагружается.

Software Watchdog (Связь): Если Slave не получает корректный пакет (с верной CRC) более 2000 мс, флаг isRemoteConnected сбрасывается. Slave переходит на чтение локальных аппаратных тумблеров.

Non-blocking Delays: Запрещено использование delay(). Все тайминги реализованы через сравнение millis() для поддержания непрерывного обмена по RS485.
