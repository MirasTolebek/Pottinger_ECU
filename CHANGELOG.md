# История изменений (Changelog) — Проект АСУ Пресс-подборщика

## [2.2] — Текущая стабильная версия (Пульт)
### Исправлено
- Устранена потенциальная уязвимость неопределенного поведения (UB) в блоке навигации меню: указатель выбора редактируемого параметра `valPtr` теперь гарантированно инициализируется безопасным адресом по умолчанию (`&edit_t_Dens`).
- Оптимизирован механизм экстренной перезагрузки внутри вектора прерывания `ISR(TIMER1_COMPA_vect)`: добавлена принудительная команда глобального запрета прерываний `cli()` перед выполнением инструкции `jmp 0`. Это обеспечивает абсолютно чистый перезапуск регистров ядра процессора ATmega328P.

## [2.1] — Текущая стабильная версия (Пресс)
### Добавлено
- Разработана и внедрена функция `updateEEPROM_Byte` для работы с внешней памятью I2C. Перед записью данных блок считывает текущую ячейку; физическая запись происходит только при несовпадении байт. Время цикла сокращено на 25–30 мс, ресурс чипа памяти защищен от износа.
- Добавлен тотальный программный валидатор конфигурационных параметров при старте `setup()`. Если память возвращает некорректное значение или мусор (`0xFF`), переменные времени автоматически перезаписываются безопасными заводскими константами.
### Исправлено
- Баг накопления ложных нажатий: переменная счетчика кликов `resetClicks` теперь принудительно сбрасывается в `0` при вызове защитной процедуры `executeEmergencyReset()`.

## [2.0] — Модернизация под бортовую сеть трактора
### Добавлено
- Реализован программный сторожевой таймер (Software Watchdog) на базе прерываний аппаратного Таймера 1 для пульта в кабине. Зависание кода дольше 4 секунд приводит к автоматическому мягкому сбросу без риска блокировки загрузчика.
- Добавлена функция циклического сохранения счетчика текущей смены (сессии) `sessionBales` во внешний EEPROM. Данные защищены от случайного выключения зажигания трактора.
- В меню конфигурации добавлен 6-й параметр: `ZASCHITA` (переменная `t_Motor`), позволяющий регулировать интервал ухода электроприводов в защитную аварию (от 5 до 30 секунд).
### Изменено
- Из кода пульта полностью удален стартовый звуковой сигнал при подаче питания. Это сделано для того, чтобы автоматический программный перезапуск устройства в поле из-за жестких наводок происходил абсолютно незаметно для слуха механизатора.

## [1.8] — Акустическое кодирование фаз
### Добавлено
- Разработан уникальный звуковой паттерн для фазы окончания обвязки и требования выгрузки рулона (`playShortLong()`). Зуммер воспроизводит комбинацию: короткий сигнал (150 мс) -> пауза (150 мс) -> длинный сигнал (800 мс). Оператор теперь безошибочно отличает команду "Открой дверь" от команды "Старт обвязки" на слух.

## [1.7] — Адаптация под силовую периферию
### Изменено
- Длительность базовых сигналов подтверждения (начало процесса, отсечка) увеличена со 150 мс до 250 мс. Данное изменение вызвано инерционностью мощных автомобильных реле и электромагнитных гудков, которые не успевали физически сдвинуть мембрану при слишком коротких импульсах тока.

## [1.5] — Оптимизация сетевого протокола
### Исправлено
- Устранена проблема "пропуска" звуковых сигналов пультом из-за сетевой задержки опроса RS485 (300 мс). Логика пульта переведена с отслеживания жестких номеров состояний пресса на детекцию самого факта изменения переменной (`currentState != prevLoopState`). Пульт теперь гарантированно перехватывает скоростные фазы работы пресса.

## [1.3] — Маршрутизация звука и отказоустойчивость
### Добавлено
- В меню пульта внедрен новый настроечный узел `ZVUK` с тремя режимами: `0` — звучит только пресс на улице; `1` — звучит только пульт в кабине; `2` — синхронный дублированный звук.
- Реализован алгоритм "живучести шины": при обрыве линии связи RS485 более чем на 2 секунды, блок пресса автоматически сбрасывает настройки маршрутизации и активирует уличный горн на максимум, предотвращая аварийную работу пресса "вслепую".

## [1.0] — Базовый стабильный релиз (MVP)
- Разделение монолитного кода на архитектуру Master-Slave.
- Запуск обмена данными по RS485 по протоколу опроса с защитой байтом четности (XOR CRC).
- Реализация конечного автомата прессования (9 логических шагов).
- Неблокирующий опрос физических кнопок с программным антидребезгом (50 мс) на базе `millis()`.
